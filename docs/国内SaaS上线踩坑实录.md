# 做一个国内SaaS有多难？FlowNote上线踩坑实录

> 从0到上线，我用了整整一个半月。不是开发太慢，是国内流程太长。
>
> 本文记录从域名注册到支付接入的完整踩坑过程，附避坑指南和时间线。

---

## 一、项目背景

**FlowNote** = AI视频笔记工具
- 粘贴B站链接 → AI提取字幕 → 生成结构化笔记
- 技术栈：Next.js + FastAPI + Whisper + DeepSeek + Celery

原以为代码写完了就能上线，没想到真正的时间黑洞是：**国内合规流程**。

## 三、逐个踩坑详解

### 🔴 第1坑：ICP备案（最耗耐心）

**预期：** 填表 → 审核 → 上线

**实际：** 填表 → 腾讯云初审（3天）→ 管局终审（10-15天）→ 工信部下发备案号

**坑点：**
- 管局给的短信核验码必须**24小时内**点击确认，错过从头来
- 备案期间网站必须**关闭访问**或显示"正在备案"，不能提前开放
- 主体信息一旦提交，后续所有认证（公安备案、微信支付）必须**完全一致**

**血泪教训：**
> 我的备案订单号：30178002009400987，等了两周。期间公安备案的数据码有效期只有30天，差点过期。

---

### 🔴 第2坑：公安联网备案（最容易忽视）

ICP备案过了≠万事大吉。公安部令第33号要求：**网站开通30天内必须完成公安备案**。

**流程：**
1. 拿到腾讯云给的备案数据码（有效期30天）
2. 登录全国互联网安全管理服务平台
3. 先提交"新增主体" → 等审核 → 再提交"新办网站"
4. 审核通过后，在网站底部加一行：
   ```
   京公网安备 XXXXXXXXXXXX号
   ```

**坑点：**
- 必须**先主体后网站**，顺序不能反
- 网站类型选错会被打回（FlowNote是工具类，选"信息咨询"或"计算机应用"）
- 是否提供互联网交互服务 → **选否**（有登录≠交互服务）
- 是否涉及管制物品 → **选否**

**血泪教训：**
> 第一次填"交互服务"选了"是"，审核员打电话问我是不是做论坛的，差点被拒。

---

### 🔴 第3坑：HTTPS配置（技术踩坑）

国内现在**没有HTTPS=不能上线**。浏览器会标红，微信支付会拒绝，微信登录会报错。

**方案对比：**

| 方案 | 难度 | 稳定性 | 适用场景 |
|---|---|---|---|
| **Caddy** | ⭐ | 自动续期 | 个人/小项目 |
| **Nginx + Certbot** | ⭐⭐⭐ | 高 | 大厂/高并发 |
| **腾讯云SSL证书** | ⭐⭐ | 手动续期 | 已有腾讯云生态 |

**踩坑实录：**
1. Caddy apt源在国外，国内下载经常超时（`dl.cloudsmith.io`连不上）
2. 解决方案：直接从GitHub Release下载二进制
3. 前端Next.js dev模式下`_next/static`资源404 → 原因是僵尸进程占用了3000-3005端口
4. 解决方案：`pkill -f "next-server"` 不能只杀 `npm run dev`

---

### 🔴 第4坑：微信支付商户号（信息不一致=全挂）

**以为的流程：** 营业执照 → 填资料 → 审核通过 → 接入支付

**实际的流程：**

```
营业执照 ──→ 经营信息 ──→ 结算账户 ──→ 管理员绑定
                              │
网站信息（必须HTTPS+ICP备案）──┤
                              │
AppID绑定（必须有已认证服务号）─┘
                              │
签约入驻 ──→ 配置API密钥 ──→ 下载证书 ──→ 配置回调URL
```

**致命坑点：**

#### 坑4.1：商户号类型限制
- 新商户号（1113595290）= 涵讯信息科技
- 旧商户号（1612137979）= 涵讯科技
- **两个商户号不能混用私钥和证书**，序列号必须和私钥一一对应

**报错：** `商户证书序列号有误。请使用签名私钥匹配的证书序列号`

#### 坑4.2：Native支付需要appid
- 微信支付APIv3的统一下单接口，`appid`是**必填项**
- 但微信开放平台网站应用的appid**不能绑定到商户号**
- 必须绑定**已认证的服务号/小程序**

**报错：** `当前商户号暂不支持关联该类型的AppID`

#### 坑4.3：服务号未认证
- 旧商户号绑定了一个服务号「外卖便宜社」
- 但这个服务号**没有通过微信认证**
- 结果：绑定时显示"已关联"，下单时报"mch_id与appid不匹配"

**报错：** `获取APPID认证信息失败，AppID未认证，请通过微信认证后重试`

#### 坑4.4：APIv2 vs APIv3
- 旧商户号原来用的是APIv2，代码写的是APIv3
- APIv3需要重新申请证书（和v2不兼容）
- APIv3密钥和v2密钥是两套系统

**血泪教训：**
> 来来回回切换了3个商户号，试了2个appid，花了整整3天才搞明白：
> **支付接口需要「已认证服务号的appid」→ 网站应用的appid不行 → 必须先花300元认证服务号。**

---

### 🔴 第5坑：微信开放平台（网站应用审核）

**预期：** 提交 → 审核 → 拿到appid → 做微信登录

**实际：**
1. 注册开放平台账号（企业认证1-3天）
2. 创建网站应用FlowNote
3. 提交《微信开放平台网站信息登记表》（打印→盖章→扫描上传）
4. 等待审核（1-7个工作日）
5. 拿到appid：**wx5e4a0971dd87700f**

**坑点：**
- 网站必须能正常访问（审核员会打开看）
- 必须已有ICP备案号
- 登记表必须**盖公章**，手写填写

---

### 🔴 第6坑：前后端联调（环境不一致）

**本地开发：** `localhost:3000` → `localhost:8001`，一切正常

**服务器部署：**
- Next.js dev模式`useSearchParams`和静态生成冲突 → **不能用`npm run build`**
- Tailwind CSS dev模式下样式丢失 → **端口被僵尸进程占用**
- SQLite并发写入锁表 → **Celery worker和FastAPI同时写会卡死**

**解决方案：**
```bash
# 重启三板斧
fuser -k 8001/tcp
find . -name "__pycache__" -exec rm -rf {} +
pkill -f "next-server"
```

## 四、避坑时间线（最优顺序）

如果你也要做国内SaaS，**按这个顺序走**，能省至少2周：

| 阶段 | 任务 | 耗时 | 注意点 |
|---|---|---|---|
| **Week 1** | 注册公司/个体户 + 买域名 | 3-5天 | 主体名后续改不了 |
| **Week 1-2** | ICP备案 | 15天 | 同时开发代码 |
| **Week 2** | 注册微信服务号 + 认证（300元） | 1-3天 | **必须在支付之前** |
| **Week 2-3** | 注册微信开放平台 + 创建网站应用 | 1-7天 | 等ICP备案号 |
| **Week 3** | 申请微信支付商户号 | 1-3天 | 用已认证服务号绑定 |
| **Week 3** | 公安备案 | 3-7天 | ICP通过后30天内必须做 |
| **Week 4** | 配置HTTPS + 部署上线 | 1-2天 | 用Caddy最省事 |

**关键顺序：**
```
公司注册 → 域名 → ICP备案 → 服务号认证 → 微信支付商户号 → 开放平台 → 公安备案
```

**不能并行的：**
- ❌ 没有ICP备案号 → 不能申请微信支付
- ❌ 没有认证服务号 → 不能用微信支付Native/JSAPI
- ❌ 没有HTTPS → 微信支付回调会被拒

**可以并行的：**
- ✅ ICP备案期间 → 写代码
- ✅ 服务号认证期间 → 申请开放平台
- ✅ 商户号审核期间 → 配置HTTPS

## 六、给后来者的建议

### 如果你打算做国内SaaS

1. **先认证服务号，再写支付代码**
   > 我代码写完了才发现服务号没认证，白白等了3天。服务号认证是支付的前置条件，不是可选项。

2. **ICP备案越早越好**
   > 备案期间网站不能开放，但代码可以继续写。别等代码写完了才想起来备案。

3. **商户号和公众号主体必须一致**
   > 同一个营业执照可以开多个商户号，但每个商户号必须绑定**同一个主体**下的公众号/小程序。

4. **APIv3的证书和私钥是一一对应的**
   > 序列号A对应私钥A，序列号B对应私钥B。混用直接报401。

5. **Caddy比Nginx更适合个人项目**
   > 自动HTTPS、自动续期、配置极简。国内下载不了apt源？直接下GitHub Release的二进制。

6. **准备好300元服务号认证费**
   > 想做微信支付，这300元是省不了的。订阅号不行，必须服务号。
