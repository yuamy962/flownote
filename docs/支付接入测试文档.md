# FlowNote 微信支付接入测试文档

> 本文档包含微信支付从配置到测试的完整流程。

---

## 一、微信支付商户平台配置（必须先完成）

### 步骤 1：设置 APIv3 密钥

**路径：** 微信支付商户平台 → 账户中心 → API 安全 → 设置 APIv3 密钥

**操作：**
1. 点击"设置密钥"
2. 短信验证管理员手机
3. 粘贴以下 32 位密钥（已为你生成）：
   ```
   a1B2c3D4e5F6g7H8i9J0k1L2m3N4o5P
   ```
4. 确认设置

> ⚠️ **密钥只显示一次，务必保存好。丢失只能重置。**

---

### 步骤 2：配置 Native 支付回调 URL

**路径：** 产品中心 → Native 支付 → 产品设置 → 支付配置

**操作：**
1. 找到"Native 支付回调链接"
2. 点击"添加"或"修改"
3. 填写：
   ```
   https://flownote.cn/api/pay/notify
   ```

---

### 步骤 3：获取证书序列号 + 下载私钥

**路径：** 账户中心 → API 安全 → 证书管理

**操作：**
1. 查看"商户API证书"列表
2. 复制**证书序列号**（一串字母数字组合，如：`34A822...`）
3. 点击"下载证书"，得到一个 `.zip` 压缩包
4. 解压后找到 `apiclient_key.pem` 文件（这是私钥）

> 记录：**证书序列号** + `apiclient_key.pem` 文件路径，后面部署要用。

## 二、服务器配置

### 1. 上传私钥文件

把 `apiclient_key.pem` 上传到服务器：

```bash
# 本地执行
ssh root@118.25.22.220 "mkdir -p /root/video-notes/backend/certs"
scp apiclient_key.pem root@118.25.22.220:/root/video-notes/backend/certs/
```

### 2. 配置 .env 文件

SSH 登录服务器，编辑 `~/video-notes/backend/.env`：

```bash
ssh root@118.25.22.220
nano ~/video-notes/backend/.env
```

**追加以下内容**（把 `YOUR_CERT_SERIAL` 替换成你的证书序列号）：

```env
# 微信支付配置
WECHAT_PAY_MCHID=1113595290
WECHAT_PAY_APIV3_KEY=a1B2c3D4e5F6g7H8i9J0k1L2m3N4o5P
WECHAT_PAY_NOTIFY_URL=https://flownote.cn/api/pay/notify
WECHAT_PAY_CERT_SERIAL=YOUR_CERT_SERIAL
WECHAT_PAY_PRIVATE_KEY_PATH=./certs/apiclient_key.pem
```

保存退出（Ctrl+O → Enter → Ctrl+X）。

## 五、测试流程

### 测试 1：查看套餐列表

1. 浏览器访问 `https://flownote.cn/pricing`
2. 应该看到 3 个套餐卡片：
   - 基础版 ¥19.00 / 300分钟
   - 专业版 ¥49.00 / 1000分钟（推荐）
   - 无限版 ¥99.00 / 不限

### 测试 2：创建订单 + 支付

1. 登录账号
2. 点击"立即购买"（建议先测基础版 ¥19）
3. 弹出二维码弹窗
4. **用微信扫码支付**
5. 支付成功后，弹窗显示"支付成功！时长已到账"

### 测试 3：验证时长到账

1. 关闭弹窗，页面顶部应显示更新后的剩余时长
2. 或进入"用户中心"查看时长是否增加

### 测试 4：查看订单记录（可选，后续可加页面）

```bash
# 服务器上查看数据库
cd ~/video-notes/backend
```

## 六、常见问题

### Q1：后端启动报错 `cryptography` 找不到
```bash
pip install cryptography
```

### Q2：支付下单报错 `商户私钥文件不存在`
检查 `apiclient_key.pem` 是否上传到 `~/video-notes/backend/certs/`

### Q3：回调验签失败
- 检查 `.env` 里的 `WECHAT_PAY_CERT_SERIAL` 是否正确
- 检查 `WECHAT_PAY_APIV3_KEY` 是否和商户平台设置的一致

### Q4：支付成功但时长没到账
- 查看后端日志：`tail -f ~/video-notes/logs/backend.log`
- 搜索 `[PayNotify]` 看回调处理情况

### Q5：二维码不显示
- 检查网络是否能访问 `api.qrserver.com`
- 或复制 `code_url` 内容手动生成二维码测试

## 七、支付流程图

```
用户点击购买
    ↓
前端 POST /api/pay/create-order
    ↓
后端创建订单 + 调用微信支付统一下单
    ↓
返回 code_url
    ↓
前端生成二维码（用户微信扫码）
    ↓
用户支付成功
    ↓
微信 POST /api/pay/notify
    ↓
后端验签 → 解密 → 更新订单状态 → 给用户加时长
    ↓
前端轮询查询到订单已支付 → 显示成功
```
