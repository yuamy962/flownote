## 三、阶段二：GPU 环境搭建（第 1 周，与后端并行）

### 3.1 HAI 实例初始化

| # | 事项 | 预计耗时 | 说明 |
|---|------|---------|------|
| 1 | 腾讯云控制台购买 HAI T4 实例 | 10 分钟 | 镜像选 Ubuntu 20.04/22.04 + GPU 驱动预装 |
| 2 | SSH 登录 HAI，确认 GPU 可用 | 10 分钟 | 运行 `nvidia-smi` 确认驱动 |
| 3 | 安装 Faster-Whisper | 30 分钟 | `pip install faster-whisper` |
| 4 | 下载 Whisper large-v3 模型 | 5~10 分钟 | 首次转录时自动下载，约 3GB |
| 5 | 编写 Whisper API 服务（FastAPI） | 2~3 小时 | 接收音频文件 → 转录 → 返回 JSON |
| 6 | 配置 HAI 自动启停脚本 | 2 小时 | 用腾讯云 API SDK，空闲 5 分钟自动关机 |
| 7 | 配置内网互通（Web 服务器 ↔ HAI） | 1 小时 | 同地域同 VPC，走内网 IP 通信 |

### 3.2 Whisper API 接口定义

```
POST /transcribe
  Body: audio_file (multipart) + model=large-v3|medium
  Response: { task_id, status: "queued" }

GET /transcribe/{task_id}
  Response: { status, segments: [{start, end, text}...] }
```

### 3.3 自动启停逻辑

```
用户提交任务 ──→ Celery Worker 检测到无 GPU 运行
                 ──→ 调用腾讯云 API 启动 HAI 实例
                 ──→ 等待 HAI 就绪（约 60~90 秒冷启动）
                 ──→ 发送任务到 HAI Whisper API
                 ──→ 转录完成，返回结果
                 ──→ 5 分钟无新任务，自动关机
```

> **关键点**：CPU 转录（备选）和 GPU 转录走同一套任务队列接口，对用户透明。GPU 未就绪时任务先排队。

## 十一、Q&A（常见疑问）

### Q1: 备案期间真能开发吗？
**能**。服务器 IP 地址可以直接访问，Nginx 配自签证书也有 HTTPS。内测用户用 IP 访问完全可行。备案只影响微信登录回调（必须域名）和公开品牌信誉。

### Q2: HAI 冷启动 60~90 秒，用户能接受吗？
**能**。用户感知流程：提交视频 →「正在处理（视频时长约 X 分钟）」→ 读条。首任务在 GPU 启动期间 CPU 走备选方案，或者前端显示「正在启动转录引擎」，用户不会觉察到 GPU 启停。

### Q3: 一个月要付多少 GPU 钱？
按 MVP 初期 50 用户、月均转录 200 小时估算：
- GPU 运行时间 ≈ 200h ÷ 15x（GPU 加速比）≈ **13.3 小时**
- GPU 费用 ≈ 13.3 × ¥6 ≈ **¥80/月**
- 加上 Web 服务器 ¥40，总固定成本 ≈ **¥120/月**

### Q4: CPU 转录备选有用吗？
**有**。作为 Fallback 方案：GPU 启动失败、GPU 配额用完、高峰期排队过长时自动降级到 CPU（直接 Web 服务器上用 Faster-Whisper CPU medium 模型）。速度慢（0.5~1x），但至少能完成任务。

### Q5: 如果备案下来了，当天能上线吗？
能。域名解析 + SSL + HTTPS 配置约 1 小时。前提是开发已完成、内测通过。这就是「备案与开发并行」的意义。
