## 一、服务器选型方案（HAI GPU 替代原腾讯云轻量方案）

### 1.1 方案变更缘由

原方案（腾讯云轻量应用服务器 4核8G，¥200~300/月）存在以下瓶颈：

| 问题 | 影响 |
|------|------|
| CPU 转录速度慢（实时因子 0.5~1.0x） | 30分钟视频需 30~60 分钟处理，用户体验差 |
| 转录成为系统瓶颈 | Web 服务与 Whisper 抢 CPU，相互影响 |
| 用户增长后扩展困难 | 单机并发上限 2~4 个任务，队列堆积快 |

**新方案采用腾讯云 HAI（High-performance Application Instance）GPU 服务器**，关键优势：

- GPU（T4）转录速度约实时因子 10~20x，30分钟视频 1.5~3 分钟处理完
- 按需启停，不用时关机不计费，成本可控
- 与 Web 服务分离，互不影响

### 1.2 HAI 方案配置

| 项目 | 规格 |
|------|------|
| 实例类型 | HAI 深度学习加速实例 |
| GPU | NVIDIA T4（16GB 显存） |
| 存储 | 50GB 系统盘 + 100GB 数据盘 |
| 计费模式 | **按量计费**（关机不计费） |

T4 运行 Whisper large-v3 模型，转录速度约 10~20x real-time，medium 模型则更快。

### 1.3 定价细节

| 配置 | 运行中价格 | 关机状态 |
|------|-----------|---------|
| T4 GPU 实例 | 约 ¥5~7/小时 | ¥0（仅磁盘占用 ¥0.01~0.02/小时） |

### 1.4 使用成本模型

#### 场景 A：按需启动（推荐 MVP 模式）

用户提交转录任务 → API 自动启动 HAI → 处理完成后保留 5 分钟无新任务自动关机。

| 月转录量 | GPU 运行时长 | GPU 费用 | 磁盘费用 | **月费用合计** |
|---------|-------------|----------|---------|-------------|
| **500 小时**（约 50 用户 × 10h） | ~25~50 小时 | ¥125~350 | ~¥15 | **¥140~365** |
| **1,000 小时**（约 100 用户 × 10h） | ~50~100 小时 | ¥250~700 | ~¥15 | **¥265~715** |
| **2,000 小时**（约 200 用户 × 10h） | ~100~200 小时 | ¥500~1,400 | ~¥15 | **¥515~1,415** |

> 按量计费模式下，成本与实际转录量成正比，**不会有空跑浪费**。

#### 场景 B：包周/包月（用户量稳定后）

当每日处理量稳定在 50+ 小时时，包月更划算。

| 时长 | 价格 | 折合每小时 |
|------|------|-----------|
| 包周 | 约 ¥300~400/周 | ¥1.8~2.4/小时 |
| 包月（竞价） | 约 ¥800~1,200/月 | ¥1.1~1.7/小时 |

### 1.5 与原方案对比

| 对比项 | 原方案（腾讯云轻量 4核8G） | 新方案（HAI T4 GPU） | 结论 |
|-------|-------------------------|---------------------|------|
| 转录速度 | 0.5~1.0x（慢） | 10~20x（快） | **新方案胜出** |
| 综合月费 | ¥200~300（固定） | ¥140~365（按需） | 起步成本接近 |
| 扩展灵活性 | 需整机升级 | 按需启停 + 更高并发 | **新方案胜出** |
| Web 服务 | 同机部署，相互影响 | 需另配轻量服务器跑 Web | 需额外 ¥30~50/月 |
| 运维复杂度 | 单机简单 | Web + GPU 两台机器 | 略增但可控 |

> **综合建议**：MVP 阶段采用 **HAI 按需启动 + 腾讯云轻量 2核2G（¥30~50/月）跑 Web** 的方案组合。总启动成本约 ¥200~400/月，比原方案更低。

### 1.6 架构调整

原单机架构调整为双机分离：

```
┌─────────────────┐     ┌──────────────────┐
│  轻量服务器      │     │  HAI GPU 实例     │
│  2核2G ¥40/月    │     │  T4 按需启停       │
│                  │     │                  │
│  Nginx           │     │  Faster-Whisper  │
│  Next.js (前端)  │     │  large-v3/medium │
│  FastAPI (后端)  │◄───►│  (API 触发)      │
│  Celery Worker   │     │                  │
│  Redis + PgSQL   │     │  (关机不计费)     │
└─────────────────┘     └──────────────────┘
```

- **Web 服务器**：轻量 2核2G，跑 Nginx + Next.js + FastAPI + Redis + PostgreSQL + Celery 调度
- **GPU 服务器**：HAI T4，跑 Faster-Whisper，由 Celery Worker 通过 API/SSH 触发任务
- **网络**：两台机器部署在同一腾讯云地域，走内网通信（免流量费）

### 1.7 自动启停策略

| 事件 | 动作 |
|------|------|
| 用户提交转录任务（无字幕需 ASR） | Celery 检测到任务 → 调用 API 启动 HAI |
| HAI 启动完成 | Worker 发送转录任务到 HAI |
| 转录完成 | HAI 返回结果给 Web 服务器 |
| 任务队列为空超过 5 分钟 | 自动关机 HAI |
| 高峰时段连续有任务 | HAI 保持运行，逐任务处理 |

> **实现方式**：通过腾讯云 API/SDK 控制 HAI 实例启停，启动约需 1~2 分钟（冷启动），可通过预热机制优化。

## 二、最终定价方案（补充修订原文档 2.4 节）

### 2.1 定价调整说明

| 基础版 | ¥9.9/5小时 | **¥19.9/600分钟（10小时）** | 与 AI好记 ¥39/600分钟 直接对标，一半价格 |

### 2.2 最终定价表

| 套餐 | 月付 | 年付（≈月付 5.5 折） | 转录额度 | 单视频上限 |

| 档位 | AI好记价格 | 我们价格 | 差额 | 我们的优势 |
|------|-----------|---------|------|----------|

> **定价公式**：¥19.9 = AI好记入门价的 **51%**；¥59 = AI好记高阶价的 **66%**。全系定价低于竞品 34%~49%。

### 2.4 各套餐盈利分析

按边际成本 ≈¥0.01/分钟（采用 HAI GPU 方案后的估算成本）计算：

| 套餐 | 价格 | 用满成本 | 单人最坏利润 | 单人典型利润（用 ~30%） |
|-----|------|---------|------------|----------------------|
| **基础版** ¥19.9 | 600分钟 | ¥6 | **+¥13.9** ✅ | **+¥17** ✅ |
| **标准版** ¥39 | 2,000分钟 | ¥20 | **+¥19** ✅ | **+¥33** ✅ |
| **高级版** ¥59 | 6,000分钟 | ¥60 | **-¥1** ⚠️ | **+¥41** ✅ |

> - 高级版在最极端情况（用户精准用满 6,000 分钟）下亏 ¥1，实际中极少发生
> - 典型用户月使用量约为额度的 20%~40%，三层均有可观利润
> - 年付用户预收全年费用，现金流更好

## 三、防滥用策略（补充原文档 3.4 节）

针对讨论中提到的「宿舍共用账号」风险，制定以下防滥用措施：

### 3.1 设备绑定策略

| 措施 | 实现方式 | 严格程度 |
|------|---------|---------|
| 同时在线设备数限制 | 每个账号最多 **3 台** 设备同时登录 | 中等 |
| 每日 IP 变更检测 | 同一账号每天切换超过 **3 个 IP** 触发风控 | 严格 |
| 设备指纹 | 浏览器/设备指纹绑定，更换设备需重新验证 | 中等 |
| 并发转录限制 | 同一账号同时只能进行 **1 个** 转录任务 | 严格 |

### 3.2 用量异常检测

| 触发条件 | 动作 |
|---------|------|
| 单个账号日使用量 > 200 分钟 | 触发人工审核，限速至 1x 实时 |
| 单个账号连续 7 天用满额度 | 发送确认邮件，要求验证身份 |
| 同一 IP 下有 3+ 账号高频使用 | 疑似宿舍共用，降级至免费队列优先级 |

### 3.3 经济学反滥用（核心手段）

相比技术封堵，**让共享行为不划算**才是最有效的策略：

| 共享人数 | 人均可用（基础版¥19.9） | 人均成本 | vs 单独购买 | 结论 |
|---------|----------------------|---------|------------|------|
| 2 人共享 | 300 分钟/人 | ¥10/人 | 单独买 ¥19.9 | 省一半，**有动机** |
| 3 人共享 | 200 分钟/人 | ¥6.6/人 | 单独买 ¥19.9 | 省 67%，但额度太少 |
| 4 人共享 | 150 分钟/人 | ¥5/人 | 单独买 ¥19.9 | 额度不够用，没意义 |
| 5 人共享 | 120 分钟/人 | ¥4/人 | 免费版 60 分钟免费 | 不如每人用免费版 |

> **结论**：最多 2 人共享有微利，3 人以上不如各自买基础版或直接用免费版。加上设备绑定和并发限制，共享成本远高于收益。
