## 二、首页

### 2.1 页面目标

让访客 10 秒内理解产品价值，引导注册或直接尝试，突出"B 站视频 → 结构化笔记"核心场景。

### 2.2 核心交互

| 元素 | 说明 |
|------|------|
| B 站链接输入框 | 未登录用户可输入，点击「立即试用」跳注册页，注册完成后自动带入链接继续 |
| 「免费开始」按钮 | 跳转注册页 |
| 功能演示区 | 短 GIF 展示：粘贴链接 → 转录 → 笔记生成 → 导出 |
| 套餐卡片 | 三档付费套餐预览，点击跳转注册/登录后进入支付 |

### 2.3 交互细节

- 输入框 Placeholder：`粘贴 B 站视频链接，如 https://www.bilibili.com/video/BVxxx`
- 输入非 B 站链接：红色提示「目前仅支持 B 站链接，其他视频请升级后上传本地文件」
- 已登录用户：输入框直接可用，点击「开始转录」跳工作台并自动提交

## 三、注册与登录

### 3.1 注册流程

```
输入邮箱 + 密码
    ↓
点击「发送验证码」→ 验证邮件发出（有效期 10 分钟）
    ↓
输入 6 位验证码 → 点击「注册」
    ↓
账号创建，自动登录，跳转工作台
（若来自首页输入框，跳工作台并预填链接）
```

**状态反馈：**

| 场景 | 提示内容 | 位置 |
|------|----------|------|
| 邮箱已注册 | 「该邮箱已注册，去登录 →」 | 输入框下方 |
| 验证码发送成功 | 「验证码已发送，注意查收垃圾箱」 | Toast |
| 验证码错误 | 「验证码不正确，请重新输入」 | 输入框下方 |
| 验证码过期 | 「已过期，请重新获取」+ 重发按钮 | 输入框下方 |
| 密码强度不足 | 「至少 8 位，需包含字母和数字」 | 输入框下方 |
| 注册成功 | 「注册成功，欢迎使用！」 | Toast，2 秒消失 |

发送验证码后按钮进入 60 秒倒计时：`重新发送 (58s)`，期间不可点击。

### 3.2 登录

- 邮箱 + 密码登录
- 连续 **5 次错误 → 要求输入图形验证码**，而非直接锁定
- 连续 **10 次错误 → 锁定 15 分钟**，提示「登录失败次数过多，请 15 分钟后再试或重置密码」
- 「忘记密码」→ 输入邮箱 → 收到重置链接（有效 30 分钟）→ 设置新密码

> **修正说明**：原方案"5次错误锁定30分钟"过于严格，容易误伤正常用户。改为"5次后验证码 + 10次后锁定15分钟"，兼顾安全与体验。

### 3.3 微信登录（备案通过后上线）

```
点击「微信登录」
    ↓
PC：弹出二维码弹窗 / 移动端：跳转微信授权页
    ↓
授权成功 → 回调 /auth/wechat/callback
    ↓
新用户：自动创建账号（可选补充邮箱）
老用户：直接登录
    ↓
跳转工作台
```

## 四、工作台（新建任务）

### 4.1 输入方式说明

工作台新建任务区分两个 Tab，**免费用户只能看到并使用 Tab 1**：

```
[ B 站链接 ]  [ 本地上传 🔒 仅付费 ]
```

免费用户点击「本地上传」Tab：

```
🔒 本地上传仅限付费用户

上传 MP4/MOV/MP3/WAV 文件，最大 2GB
支持 GPU 快速转录，3-6 分钟出结果

[升级到基础版 ¥15/月]   [了解套餐]
```

### 4.2 Tab 1：B 站链接输入

**操作步骤：**

1. 粘贴 B 站视频 URL
2. 自动解析视频信息（调 B 站 API，约 1~2 秒）
3. 展示视频卡片（标题、时长、封面、UP 主）
4. 根据用户套餐展示转录方式预告（见下表）
5. 确认后点击「开始转录」

**字幕检测 + 用户分层展示：**

| 用户套餐 | 检测到字幕 | 未检测到字幕 |
|----------|-----------|-------------|
| 免费版 | ✅ 蓝标「有字幕，约 30 秒出结果」 | ⚠️ 橙色提示「无字幕，将使用 CPU 转录，预计 30-60 分钟」+ 升级引导 |
| 基础版/以上 | ✅ 蓝标「有字幕，约 30 秒出结果」 | ✅ 绿标「无字幕，GPU 快速转录，预计 3-6 分钟」 |

**免费用户无字幕时的升级引导：**

```
⚠️ 该视频没有内嵌字幕，将使用 CPU 转录
   预计等待时间：30-60 分钟

   升级基础版（¥15/月）可享受 GPU 快速转录，3-6 分钟出结果
   [升级加速]   [继续慢速转录]
```

点击「继续慢速转录」后，按钮文案变为「我已知晓，开始转录（预计 30-60 分钟）」，需二次确认。

**视频解析异常状态：**

| 场景 | 提示文案 |
|------|----------|
| 解析中 | 输入框右侧 Loading + 「正在获取视频信息...」 |
| 链接无效 | 「视频不存在或链接有误，请检查后重试」（红色） |
| 视频需要 B 站登录 | 「该视频为 B 站会员专属或私密视频，无法解析。您可以尝试其他公开视频，或升级付费版上传本地文件」 |
| 时长超套餐上限 | 「视频时长 X 分钟，超出当前套餐单视频上限（Y 分钟）」+ 升级按钮 |
| 本月时长配额不足 | 「本月剩余时长 X 分钟，处理该视频需 Y 分钟」+ 两个选项：升级 / 等待下月重置 |

> **修正说明**：B站会员视频的处理增加了"尝试其他公开视频"的替代路径，避免免费用户陷入死胡同。

### 4.3 Tab 2：本地文件上传（付费用户）

支持格式：MP4、MOV、MP3、WAV，最大 2GB

**操作步骤：**

1. 拖拽文件或点击「选择文件」
2. 前端解析文件时长与大小，展示文件卡片
3. 展示转录通道预告：「将使用 GPU 快速转录，预计 3-6 分钟」
4. 点击「开始转录」

**上传状态：**

| 场景 | 展示方式 |
|------|----------|
| 拖拽悬停 | 上传区蓝色高亮，「松开即可上传」 |
| 上传中 | 进度条 + 百分比，支持取消 |
| 格式不支持 | 「不支持该格式，请上传 MP4/MOV/MP3/WAV」（红色） |
| 文件超出 2GB | 「文件大小超出限制（最大 2GB）」（红色） |
| 上传成功 | 文件卡片展示：文件名、大小、预估时长 |
| 上传失败 | 「上传失败，请检查网络后重试」+ 重试按钮 |

### 4.4 提交任务

点击「开始转录」后：

1. 前端校验配额是否充足
2. 校验通过 → 提交后端，获取 `task_id`
3. 自动跳转任务处理页 `/dashboard/tasks/:id/processing`

```
❌ 处理失败

[失败原因]

本次消耗的时长配额已自动退还
[重新提交]   [返回工作台]
```

**失败原因文案：**

| 失败类型 | 展示文案 |
|----------|----------|
| 音频质量过低 | 「音频质量不足（可能为纯音乐或严重噪音），无法识别，建议更换视频」 |
| 无有效音轨 | 「未检测到语音内容，请确认视频包含讲解音频」 |
| GPU 启动失败 | 「GPU 服务暂时不可用，已自动重新排队，时长配额不扣除」 |
| 转录超时 | 「转录任务超时，时长配额已自动退还，请稍后重试」 |
| 视频被删除 | 「该 B 站视频已被删除或设为私密，无法处理」 |
| 系统错误 | 「系统处理出错，时长配额已自动退还，若多次失败请通过用户中心反馈」 |

> **修正说明**：删除"GPU 月预算超限"的降级文案。付费用户**永不降级至 CPU**，GPU 预算超限仅影响免费用户的排队时长（队列变长）。

### 5.3 HAI 冷启动感知设计原则

HAI GPU 在低谷期（00:00-18:00）空闲 5 分钟后自动关机，高峰期（18:00-24:00）常开。用户交互层面：

| 时间段 | GPU 状态 | 用户感知 |
|--------|----------|----------|
| 18:00-24:00（高峰期） | 常开 | 无感，提交即处理 |
| 00:00-18:00，有任务 | 冷启动中 | 显示「GPU 启动中，约 60-90 秒」进度提示 |
| 00:00-18:00，运行中 | 已热机 | 无感 |

冷启动期间页面展示启动进度条（伪进度，90 秒跑完），避免用户误以为卡死。

## 六、任务结果页

### 6.1 页面布局

左右两栏（移动端改为 Tab 切换）：

```
┌───────────────────────────────────────────────────────┐
│  [← 返回]  视频标题                  [导出 Markdown ▼] │
│  时长 · GPU转录 / CPU转录 / 字幕解析 · 完成时间         │
├──────────────┬────────────────────────────────────────┤
│              │  [转录原文]  [AI 总结]  [结构笔记]       │
│  视频封面    │                                        │
│  + 基本信息  │  内容区（可编辑）                       │
│              │                                        │
└──────────────┴────────────────────────────────────────┘
```

顶部标签栏显示转录路径，帮助用户了解本次处理方式：

- `⚡ GPU 快速转录` — 付费用户走 HAI GPU
- `🔤 字幕解析` — 有字幕直接解析
- `🖥 CPU 标准转录` — 免费用户无字幕通道

### 6.2 Tab 1：转录原文

- 按段落展示，每段开头带时间戳（如 `[02:35]`）
- 时间戳点击：跳转 B 站对应时间点（新开标签页）
- 本地上传的视频：时间戳仅展示，不可跳转
- 全文搜索：页面内搜索框，关键词高亮
- 内容区固定高度独立滚动，Tab 栏常驻顶部

### 6.3 Tab 2：AI 总结

卡片式展示，包含：

- **核心观点**（3-5 条，带序号）
- **章节摘要**（按时间段分节，标注时间范围）
- **关键词**（高亮色标签）
- **适合人群 & 学习建议**

### 6.4 Tab 3：结构笔记

- 默认 Markdown 渲染视图
- 右上角「预览 / 源码」切换
- 源码模式支持在线编辑，防抖 1 秒自动保存
- 保存状态提示：「已自动保存」/ 「保存中...」/ 「保存失败，请重试」

**笔记格式切换（MVP 暂不实现，二期补充）：**

> **修正说明**：原方案"每任务限 3 次重新生成"增加了 LLM API 成本和实现复杂度。MVP 阶段固定生成一次，笔记格式切换（大纲式/康奈尔式）延后到二期。

### 6.5 导出交互

点击「导出 Markdown ▼」展开选项：

```
导出内容（可多选）：
☑ 转录原文
☑ AI 总结
☑ 结构笔记
──────────────────
  📥 下载 .md 文件
  📋 复制到剪贴板
```

- 下载：浏览器触发下载，文件名「视频标题_笔记.md」
- 复制：Toast 提示「已复制，粘贴到 Notion / Obsidian 即可使用」

**免费版导出限制（修正后）：**

免费版可导出全部内容（转录原文 + AI 总结 + 结构笔记），但**每月限 3 次导出**，超出后：

```
🔒 本月导出次数已用完（3/3）

付费版每月不限导出次数

[升级解锁无限导出]   [仅复制到剪贴板（不限次）]
```

> **修正说明**：原方案"免费版仅导出结构笔记"过于严格，用户无法验证转录质量。改为"限导出次数 + 不限复制"，既控制成本，又让用户完整体验产品价值。

## 九、订阅与支付

### 9.1 定价页

四列套餐卡片，专业版加「推荐」角标。

**月付 / 年付切换：**

## 十、全局交互规范

### 10.1 顶部导航栏

常驻显示当前用户时长余额（登录后）：

```
[Logo]  工作台  历史记录       ⏱ 剩余 42 分钟  [头像▼]
```

| 余额状态 | 颜色 | 附加操作 |
|----------|------|----------|
| 充足（>30 分钟） | 默认色 | 无 |
| 偏少（10-30 分钟） | 橙色 | 无 |
| 不足（<10 分钟） | 红色 | 显示「升级 →」快捷入口 |
| 耗尽 | 红色加粗 | 显示「升级解锁」按钮 |

### 10.2 登录态管理

Token 过期时，用户下次操作弹出「登录已过期，请重新登录」弹窗，登录完成后返回原页面，不强制跳转。

### 10.3 通用错误处理

| 错误类型 | 展示方式 | 文案 |
|----------|----------|------|
| 网络断开 | 顶部橙色 Banner | 「网络已断开，正在重连...」 |
| 服务器错误（5xx） | 页面级错误状态 | 「服务暂时不可用，请稍后刷新重试」 |
| 无权限（401） | 弹窗 | 「请先登录」|
| 资源不存在（404） | 独立 404 页 | 「页面不存在，可能已被删除」 |
| 操作频繁（429） | Toast | 「操作太频繁，请稍后再试」|
| GPU 资源紧张 | 处理页 Banner | 「当前转录任务较多，您的任务仍在排队中，请耐心等待」 |

> **修正说明**：删除"GPU 月预算超限，付费用户不受影响"的文案。付费用户**始终享有 GPU 通道**，资源紧张时仅表现为排队时间略长，不会降级。

### 10.4 移动端适配

- 工作台：链接输入框全宽；上传区点击触发系统文件选择器
- 任务处理页：进度展示垂直居中，信息简化
- 任务结果页：三个 Tab 顶部切换（转录原文 / AI 总结 / 结构笔记）
- 支付：微信支付自动唤起微信 App（H5 支付）
- 底部固定导航：工作台 / 历史 / 用户中心

## 十一、关键用户旅程

### 旅程 A：新用户首次使用（B 站视频有字幕）

```
访问首页 → 输入 B 站链接 → 点击「立即试用」
    ↓
注册（约 1 分钟）→ 邮箱验证
    ↓
返回工作台，链接已预填
    ↓
解析视频（1-2 秒）：显示「✅ 有字幕，约 30 秒出结果」
    ↓
点击「开始转录」→ 处理页：字幕解析约 10-30 秒 → AI 生成约 10-20 秒
    ↓
结果页：查看笔记 → 导出 Markdown

全程约 3-5 分钟，体验流畅，转化关键点：看到笔记质量后引导升级
```

### 旅程 B：免费用户遇到无字幕视频

```
提交 B 站链接 → 检测无字幕
    ↓
展示提示：「无字幕，CPU 转录预计 30-60 分钟」+ 升级引导
    ↓
用户选择 A：点击「升级加速」→ 定价页 → 基础版支付 → 返回继续处理（GPU 通道）
用户选择 B：点击「继续慢速转录」→ 二次确认 → 进入排队

    选择 B 进入处理页：
    - 展示 CPU 进度 + 预计时长
    - 主动推荐允许浏览器通知或邮件通知
    - 完成后 Toast + 通知：「您的笔记已生成，感谢您的耐心」+ 升级引导
```

### 旅程 C：付费用户提交本地视频（非高峰期）

```
登录（付费版）→ 工作台 → 「本地上传」Tab
    ↓
上传 MP4 文件（1 小时课程录播）
    ↓
提示：「GPU 快速转录，预计 3-6 分钟」
    ↓
点击「开始转录」
    ↓
00:00-18:00 低谷期：GPU 处于关机状态
    → 处理页显示「GPU 启动中，约 60-90 秒后开始...」（伪进度条）
    ↓
GPU 就绪 → 转录开始（large-v3 模型，约 4 分钟）→ AI 生成（约 20 秒）
    ↓
结果页：查看 → 在线编辑补充 → 导出 Markdown → 导入 Obsidian
```

### 旅程 D：时长配额不足时的引导路径

```
提交视频 → 校验：本月剩余 5 分钟，该视频需 45 分钟
    ↓
展示提示：
  「本月剩余时长不足（5/45 分钟）」
  [升级套餐，立即获得更多时长]（主要按钮）
  [等待下月重置（X 天后）]（次要链接）
    ↓
点击「升级套餐」→ 定价页（当前套餐的下一档高亮）→ 支付
    ↓
支付成功 → 自动返回工作台 → 重新提交任务
```

## 十二、升级引导出现时机汇总

以下场景需出现升级引导，引导文案应明确说明升级后的**具体收益**：

| 触发场景 | 引导文案要点 |
|----------|-------------|
| 免费用户访问「本地上传」Tab | 「解锁上传功能，支持 GPU 快速转录」 |
| 免费用户遇到无字幕视频 | 「升级后 3-6 分钟出结果，无需等待 30-60 分钟」 |
| 免费用户 CPU 转录完成 | 「下次升级，3-6 分钟出结果」（完成后 + 感谢语） |
| 免费用户导出次数用完 | 「升级解锁无限导出 + 剪贴板复制不限次」 |
| 时长配额不足 | 「升级获得更多时长，立即继续」 |
| 顶部余额不足 10 分钟 | 导航栏红色余额 + 「升级 →」入口 |

> **修正说明**：删除"结果页停留 60 秒弹窗"。MVP 阶段简化为固定横幅或自然触发点，不增加"每用户只弹 1 次"的后端复杂度。
