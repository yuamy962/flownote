# 视频笔记工具 — 产品需求文档 v3.0（HAI 双机最终版）

> 版本：v3.0  
> 目标用户：在校大学生、考研/考证/技术学习者  
> 核心定位：低成本 GPU 加速转录 + AI 结构化笔记工具  
> 技术架构：**Web 服务器（2核4G）+ HAI GPU（T4 按量计费）双机分离**  
> 文档状态：基于 v2 修正版 + HAI 双机架构评审后的最终定稿

## 二、技术架构详解

### 2.1 Web 服务器（轻量应用服务器）

| 配置项 | 规格 | 月费 | 说明 |
|--------|------|------|------|
| CPU | 2核 | — | 跑 FastAPI + Next.js SSR |
| 内存 | **4G** | — | **修正：v2 原方案 2G 会 OOM** |
| 带宽 | 5Mbps | — | 够用 |
| 磁盘 | 80GB SSD | — | 系统 + 代码 + 日志 |
| **费用** | — | **¥60-80/月** | 包年折后 |

**部署组件：**
- Next.js 前端（PM2 进程管理）
- FastAPI 后端（Supervisor 进程管理）
- PostgreSQL 16
- Redis 7
- Nginx
- Celery Worker（调度器，不执行重计算）
- Faster-Whisper **medium 模型**（仅处理免费版 CPU 任务）

### 2.2 HAI GPU 服务器（按量计费）

| 配置项 | 规格 | 说明 |
|--------|------|------|
| GPU | NVIDIA T4 | 16GB 显存 |
| CPU | 8核 | 辅助预处理 |
| 内存 | 32G | 大模型需要 |
| 模型 | Faster-Whisper **large-v3** | 准确率最高 |
| **单价** | **¥6-8/小时** | 按实际运行时长计费 |

**接入方式：**
- 同地域 VPC 内网互通（延迟 < 1ms）
- FastAPI 通过 HTTP 内网 IP 调用 HAI 上的 Whisper API 服务
- 不对公网暴露 HAI，仅内网通信

### 2.3 用户分层转录策略（核心修正点）

| 用户类型 | B站有字幕 | B站无字幕 | 本地上传 |
|---------|-----------|-----------|---------|
| **免费版** | ✅ 秒出，零成本 | ⚠️ CPU medium 慢速（30-60分钟） | ❌ 不支持 |
| **基础版 ¥15** | ✅ 秒出 | ✅ GPU large-v3 快速（3-6分钟） | ✅ GPU 快速 |
| **专业版 ¥29** | ✅ 秒出 | ✅ GPU 优先队列 | ✅ GPU 优先 |
| **无限版 ¥39** | ✅ 秒出 | ✅ GPU 优先 + 4小时上限 | ✅ GPU 优先 |

> **修正逻辑**：
> - 免费用户只消耗字幕解析成本（几乎为零），无字幕视频走 CPU 慢速体验差，天然形成"升级付费"的转化动力
> - 付费用户的 GPU 成本被 ¥15/月的订阅费覆盖，单位经济模型成立

### 2.4 HAI 启停策略（修正版）

```
┌──────────────────────────────────────────────────────────────┐
│                    HAI GPU 状态机                             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [关机] ──付费任务到达──→ [冷启动 60-90s] ──就绪──→ [运行中]   │
│     ↑                                              │         │
│     └──────── 高峰期保持开机 18:00-24:00 ──────────┘         │
│                                                              │
│  [运行中] ──低谷期(00:00-18:00)且无任务5分钟──→ [关机]       │
│                                                              │
│  [运行中] ──队列中任务数>3──→ [强制保持开机，直到队列清空]    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**修正说明（相比原 HAI 方案）：**
- 原方案：任何时段都是"5分钟无任务关机"
- **修正后**：高峰期（18:00-24:00）**保持常开**，避免学生用户晚课/晚自习时段频繁冷启动
- 低谷期才启用自动关机，减少碎片化空转成本

## 三、功能需求

### 3.1 输入方式

| 输入方式 | 优先级 | 免费版 | 付费版 | 说明 |
|----------|--------|--------|--------|------|
| B 站视频链接 | P0 | ✅ | ✅ | 优先解析 CC 字幕（70%视频有字幕） |
| 本地文件上传 | P0 | ❌ | ✅ | MP4/MOV/MP3/WAV，最大 2GB |
| 百度网盘 | P1 | — | — | OAuth 导入（二期） |
| 阿里网盘 | P1 | — | — | OAuth 导入（二期） |

### 3.2 核心功能

#### 3.2.1 语音转文字

- **B 站字幕优先**：检测 CC 字幕，有则直接解析，秒出结果
- **免费版无字幕**：走本地 CPU medium 模型，明确提示"预计等待 30-60 分钟"
- **付费版无字幕/上传**：走 HAI GPU large-v3，提示"预计等待 3-6 分钟"
- **时间戳对齐**：段落级时间戳，点击跳转视频位置
- **质量过滤**：信噪比过低、纯音乐视频自动拒绝

#### 3.2.2 AI 总结与笔记

- 视频摘要（3-5 条核心观点）
- 章节分段摘要
- 关键词高亮
- 结构化笔记（大纲式 / 康奈尔式可选）
- Markdown 一键导出（兼容 Notion/Obsidian/Typora）

#### 3.2.3 视频问答（P1）

- 基于转录文本的 RAG 问答
- 回答引用时间戳
- 专业版/无限版专属

### 3.3 用户系统

- 邮箱注册 + 验证
- 微信登录（备案通过后接入）
- JWT 鉴权（Access 7天 / Refresh 30天）
- 历史记录保留 90 天，支持软删除后 7 天内恢复

### 3.4 订阅与支付

#### 定价方案（最终版）

| 套餐 | 价格 | 转录时长/月 | B站字幕 | 无字幕/上传 | 导出 | 问答 |
|------|------|-------------|---------|-------------|------|------|
| 免费版 | ¥0 | 60 分钟 | ✅ 秒出 | ⚠️ CPU慢速 | 基础Markdown | ❌ |
| 基础版 | **¥15/月** 或 ¥128/年 | 5 小时 | ✅ 秒出 | ✅ GPU快速 | 完整导出 | ❌ |
| 专业版 | **¥29/月** 或 ¥248/年 | 20 小时 | ✅ 秒出 | ✅ GPU优先 | 完整导出 | ✅ |
| 无限版 | **¥39/月** 或 ¥328/年 | 50 小时 | ✅ 秒出 | ✅ GPU优先 | 完整导出 | ✅ |

> **定价修正说明**：
> - 基础版从 ¥9.9 提到 **¥15**，覆盖 GPU 碎片化成本 + 为后期涨价留空间
> - 专业版 ¥29（对标飞书妙记 ¥28，但功能更多）
> - 无限版 ¥39 设置 50 小时"公平使用"上限，防止脚本滥用

#### 支付方式

- 微信支付（扫码 + JSAPI）
- 支付宝（二期补充）
- 手续费 0.6%

#### 退款策略

- 转录失败 → 自动退还本次消耗时长
- 订阅 7 天内未使用 → 全额退款
- 年付中途退订 → 按月比例退款

## 四、非功能需求

### 4.1 性能指标

| 场景 | 指标 |
|------|------|
| B站视频有字幕 | ≤ 30 秒出结果 |
| 付费版 GPU 转录（30分钟视频） | ≤ 3 分钟 |
| 免费版 CPU 转录（30分钟视频） | 15-30 分钟（明确告知） |
| 页面首屏加载 | ≤ 2 秒 |
| API 响应（非转录类） | ≤ 500ms |

### 4.2 安全与风控

- 密码 bcrypt 加密
- 原视频 24h 删除，转录结果 90 天清理
- HTTPS 全站
- API 限流：
  - 普通接口：60 次/分钟/IP
  - 转录提交：5 次/小时（免费），10 次/小时（付费）
  - 注册：5 次/小时/IP + 图形验证码
- **反滥用**：
  - 设备指纹绑定（同设备多账号共享上限）
  - 行为检测（批量相似视频触发审核）
  - 无限版单月超 50 小时次月降级

## 八、成本测算（修正后真实模型）

### 8.1 固定成本（月度）

| 项目 | 金额 | 说明 |
|------|------|------|
| Web 服务器（2核4G） | ¥60-80 | **修正：2G 会 OOM** |
| 域名 | ¥4-7 | 均摊 |
| 微信认证 | ¥25 | 均摊 |
| COS 存储 + 流量 | ¥20-40 | 音频临时存储 + 备份 |
| **固定合计** | **¥109-152/月** | — |

### 8.2 变动成本（真实测算）

| 项目 | 单价 | 说明 |
|------|------|------|
| HAI GPU T4 | **¥6-8/小时** | 按实际运行时长计费 |
| DeepSeek LLM | **≈¥0.04/小时音频** | 总结 + 笔记 |
| GPU 碎片化损耗 | **×1.3 系数** | 启停空转、预热等待 |

> **修正说明**：原 HAI 方案未计入碎片化损耗，实际 GPU 计费时间比"音频时长÷加速比"多 30% 左右。

### 8.3 单用户成本与毛利

假设月度活跃用户构成：
- 50 免费用户：每人 1 小时（70%字幕 + 30% CPU）
- 20 基础版（¥15）：每人 5 小时（70%字幕 + 30% GPU）
- 10 专业版（¥29）：每人 20 小时（70%字幕 + 30% GPU）

**资源消耗测算：**

| 用户群 | 数量 | 总时长 | 字幕(70%) | GPU(30%) | GPU处理时间 | 碎片化后 | GPU费用 |
|--------|------|--------|-----------|----------|-------------|----------|---------|
| 免费 | 50 | 50h | 35h(¥0) | 15h(CPU) | — | — | **¥0** |
| 基础版 | 20 | 100h | 70h(¥0) | 30h | 30÷15=2h | 2.6h | **¥15.6** |
| 专业版 | 10 | 200h | 140h(¥0) | 60h | 60÷15=4h | 5.2h | **¥31.2** |
| **合计** | **80** | **350h** | **245h** | **105h** | **—** | **7.8h** | **¥46.8** |

**收入与利润：**

| 项目 | 金额 |
|------|------|
| 基础版收入 | 20 × ¥15 = **¥300** |
| 专业版收入 | 10 × ¥29 = **¥290** |
| **总收入** | **¥590** |
| GPU 成本 | ¥46.8 |
| LLM 成本 | 350h × ¥0.04 = ¥14 |
| 固定成本 | ¥130（取中位数） |
| **总成本** | **¥190.8** |
| **毛利** | **¥399.2** |
| **毛利率** | **67.7%** |

> **结论**：单位经济模型成立。即使免费用户占比更高，只要付费转化率 ≥ 5%，即可覆盖固定成本。
