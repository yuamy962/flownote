## 三、扣费规则

### 双轨时长体系
```
总可用时长 = 订阅时长(monthly) + 永久时长(permanent)
```

| 类型 | 来源 | 有效期 | 扣费优先级 |
|:---|:---|:---|:---|
| **订阅时长** | 购买月卡/年卡获得 | 按套餐有效期（30天/365天）| 优先扣除 |
| **永久时长** | 初始赠送60分钟 + 邀请奖励 | 永久有效 | 订阅不足时扣除 |

### 扣费顺序
1. 如果用户是 **pro_year / pro / basic_year / basic** 套餐且未过期 → 先扣 `monthly_minutes`
2. `monthly_minutes` 不足 → 再扣 `permanent_minutes`
3. 两者都不足 → 拒绝创建任务（402 错误）

### 创建任务时扣费
- B站视频：解析后按 `ceil(duration / 60)` 预扣分钟
- 本地上传：先扣 1 分钟保底，转录完成后按实际时长补扣/返还

## 五、到期与续费机制

### 到期提醒
- 到期前 **3 天**：发送提醒通知（预留接口，可接入微信模板消息）
- 到期后 **48 小时**：未续费则自动降级为 `free`，`monthly_minutes` 清零

### 续费逻辑
- 用户购买新套餐 → 覆盖现有 `plan` 和 `monthly_minutes`
- 从购买时间起重新计算有效期（月卡+30天，年卡+365天）

## 六、数据模型

### Plan 表
```python
id              String(20) PK     # basic / pro / basic_year / pro_year
name            String(50)
price_cent      Integer           # 单位：分
duration_minutes Integer          # 套餐额度
validity_days   Integer           # 有效期天数（30 / 365）
description     Text
sort_order      Integer
```

### User 表（时长相关）
```python
plan                String(20)      # 当前套餐ID
plan_expires_at     DateTime        # 套餐到期时间
monthly_minutes     Integer         # 订阅时长（有效期内可用）
permanent_minutes   Integer         # 永久时长（永不重置，初始40）
auto_renew          Boolean         # 是否开启到期提醒续费
```
